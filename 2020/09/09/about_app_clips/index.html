關於 App Clips 基本知識

Author: Ethan Date: September 9, 2020  12:43:27 Category: 技術 本文已被閱讀 次

是Apple在2020的WWDC中所推出的新架構，為的是讓使用者能快速體驗App的主要功能但不需要完整下載App的新流程。

相關技術的比較

這篇文章有寫到相關技術的比較，因此我就不在這邊贅述了

實作App Clips的相關須知

  • 是一個輕量型app，蘋果規定大小必須在10mb以內

  • 喚醒App Clips可透過 QRCode、NFC、App Clip Codes (Apple年底會推出的，結合NFC和QRCode的東西）、Safari Links、Message Links、Maps、Spotlight、App Library

  • 隱私權授權方式的限制和調整（ 不能請求敏感性資料，這裡所說的敏感性資料包含健康、運動、通訊錄、訊息、照片、文件等個人資訊 ，8小時內的通知 以及 一次性的定位授權則不需要授權）

  • App Clips中不支持使用Universal Link

  • 當使用者使用了App Clips之後，下載了主要程式，已經allow的權限會轉移到主程式，而儲存的資料，也會透過app group的id來轉移（包含透過Apple Sign In的資料也會）

  • 若要鼓勵使用者下載主要程式可以使用SKOverlap這個API

  • 如果使用者下載的App Clips 一段時間沒有使用 的話，系統就會刪除掉

  • App Clips的下載連結其實是一個Universal Link，可以在AppleConnect上面設定，也可以帶參數來呈現不同的內容及不同的頁面（需要注意的是，主程式中也需要在AppDelegate中的handleUserActivity中處理這些Universal Link的參數，不然當使用者下載主程式之後，只會打開app，不會因為參數而帶到其他頁面）

  • 設定好Universal Link之後，在Safari、iMessage、Siri建議上面會出現縮圖，縮圖以及點擊縮圖的內容，也需要在AppleConnect中設定，上傳的圖片、title以及subtitle的設定上有 一定的規範

    • 主要圖片：要呈現相關主題的圖片，而不是文字或是廣告；3:2 aspect fill的模式；2000px X 3000px的尺寸；建議使用png或jpg； 不能有透明度
    • title和subtitle：要呈現商家訊息而不是app的訊息；可以考慮作localized來支持不同的語言； title最多是18個字元而subtitle最多呈現43個字元 。
    • action：只有Apple提供的action可以使用，就如Open、View以及Play

  • App Clips的生命週期是由系統來接管的。

如何在Server端設置App Clips？

設置的方式與Universal Link的設置相同…

  1. 在Server端添加 Apple App Site Association file

  2. 在 Apple App Site Association file 中增加下圖數值

  3. 在Xcode中增加 Associated Domains Entitlement

handle App Clips連結的method名稱

  • SwiftUI：onContinueUserActivity(_:perform:)
  • App Delegate app：application(_:continue:restorationHandler:)
  • Scene-Based app：scene(_:willContinueUserActivityWithType:)
